# Moka TCP 使用备忘录

适用日期：2026-05-27

## 1. 启动顺序

在项目根目录执行：

```bash
sudo g++ -o test.out wangye.cpp axis.cpp axis.h general_6s.cpp general_6s.h struct_define.h libethercat.a -lrt -lpthread -std=c++11
sudo ./test.out
```

另开一个终端，进入 `app` 目录执行：

```bash
python3 app.py
```

网页地址：

```text
http://0.0.0.0:9002
```

## 2. 使用前确认

每次打开网页后，先连接设备，再看实时状态是否正常刷新。

如果刚启动时网页里的关节角显示全是 `0`，不要直接信任输入框默认值。

先做这两件事：

1. 点击“回填当前关节角”
2. 点击“回填当前 TCP 位姿”

如果需要，先在关节控制里执行一次“六轴归零”，确认实物已经回到零位。

## 3. 关节控制规则

当前版本已经修正了 `J3/J4/J6` 的方向。

现在网页中输入正角度时，机械臂会按标定后的正确方向运动。

如果某次开机后某个关节看起来没有回零，先用关节控制做小角度测试，再执行归零，不要直接先跑 TCP。

## 4. TCP 控制固定规则

后续 TCP 控制一律按下面的姿态使用：

```text
RX = -3.14
RY = 0
RZ = 0
```

重点：

- 只修改 `X/Y/Z`
- `RX/RY/RZ` 固定为上面这组值
- 不要手动改成 `0, 0, 0`

原因：

如果把姿态改成 `0,0,0`，机械臂腕部会额外翻转，可能出现末端姿态变化和大角度跳动。

## 5. 当前可用参考位姿

当前验证通过的末端参考位置约为：

```text
X = 470
Y = -0.5
Z = 816.45
RX = -3.14
RY = 0
RZ = 0
```

说明：

- `470, -0.5, 816.45` 是当前零位附近的末端位置参考
- 实时状态里 `RX` 在 `3.14` 和 `-3.14` 间跳动属于姿态表示边界现象
- 实际使用时统一固定填 `RX = -3.14`

## 6. 推荐操作方式

每次使用 TCP 时，按下面顺序：

1. 点击“回填当前 TCP 位姿”
2. 把 `RX` 手动改成 `-3.14`
3. 确认 `RY = 0`、`RZ = 0`
4. 只改 `X/Y/Z`
5. 执行空间指令

## 7. 推荐测试顺序

先做小步移动，确认一切正常后，再走更大范围。

第一步建议：

```text
X = 480
Y = -0.5
Z = 816.45
RX = -3.14
RY = 0
RZ = 0
```

再逐步测试：

```text
X = 470

RX = -3.14
RY = 0
RZ = 0
```

```text
X = 470
Y = -0.5
Z = 806.45
RX = -3.14

RZ = 0
```

最终再测试目标点：

```text
X = 500
Y = 0
Z = 800
RX = -3.14
RY = 0
RZ = 0
```

## 8. 异常时的处理

如果出现以下情况，先停止继续发 TCP 指令：

- 某个关节方向再次不对
- 关节执行后实物没有到对应位置
- TCP 接近目标点时腕部又出现异常大幅旋转
- 网页实时状态不刷新

处理顺序：

1. 先用关节控制做小角度测试
2. 确认六轴方向正常
3. 再执行一次关节归零
4. 然后再回到 TCP 控制

## 9. 一句话结论

以后 TCP 就按这个规则用：

```text
只改 X/Y/Z，姿态固定 RX=-3.14，RY=0，RZ=0
```
