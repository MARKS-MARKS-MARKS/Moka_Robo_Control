## 三、修改文件

### `app/static/index.html`

修改位置集中在写字功能相关代码区域。

#### 1. 新增 `hanziDataCache`

新增：

```js
const hanziDataCache = new Map();
```

作用：

- 缓存已经读取过的单字 JSON。
- 避免重复 `fetch` 同一个字符。
- 缺字时也缓存为 `null`，避免重复请求失败文件。

#### 2. 保留并重命名原硬编码笔画

原 `yongStrokes` 被改为：

```js
const yongFallbackStrokes = [...]
```

原 `fallbackStrokes` 被保留为全局常量。

作用：

- 当本地 JSON 缺失或加载失败时，仍能生成兜底轨迹。
- “永”字加载失败时仍能使用原来的硬编码笔画。

#### 3. 新增 `loadHanziData(char)`

新增异步函数：

```js
const loadHanziData = async (char) => {
    ...
    const resp = await fetch(`./hanzi-data/${encodeURIComponent(char)}.json`, { cache: 'no-store' });
    ...
};
```

作用：

- 从 `app/static/hanzi-data/{字}.json` 加载单字数据。
- 校验数据中是否存在 `medians`。
- 加载失败时返回 `null`。

#### 4. 新增 `normalizeMedians(medians)`

新增函数：

```js
const normalizeMedians = (medians) => {
    ...
};
```

作用：

- 将 hanzi-writer-data 的原始 median 坐标归一化到以字符中心为基准的范围。
- 输出格式与原有硬编码笔画格式一致：

```js
[
  [[x, y], [x, y]],
  [[x, y], [x, y]]
]
```

说明：

- 当前阶段只做基础归一化。
- 更精细的 Y 轴修正、点距重采样、点数限制属于后续阶段。

#### 5. 新增 `getWritingStrokes(char)`

新增异步函数：

```js
const getWritingStrokes = async (char) => {
    const data = await loadHanziData(char);
    if(data) return { strokes: normalizeMedians(data.medians), source: 'hanzi-data' };
    if(char === '永') return { strokes: yongFallbackStrokes, source: 'fallback' };
    return { strokes: fallbackStrokes, source: 'fallback' };
};
```

作用：

- 优先使用本地 JSON 数据。
- 失败时回退到原硬编码笔画。
- 返回 `source`，用于提示是否使用了兜底笔画。

#### 6. `makeWritingPath()` 改为异步

修改前：

```js
const makeWritingPath = () => { ... }
```

修改后：

```js
const makeWritingPath = async () => { ... }
```

主要变化：

- 对输入文字逐字调用 `await getWritingStrokes(ch)`。
- 如果某个字符缺少 JSON，则记录到 `missingChars`。
- 缺字时显示提示：

```js
window.showToast(`缺少字库数据，已使用兜底笔画：${...}`, "error");
```

#### 7. `previewWriting()` 改为异步

修改前：

```js
window.previewWriting = () => {
    const { points, raster } = makeWritingPath();
    ...
};
```

修改后：

```js
window.previewWriting = async () => {
    const { points, raster } = await makeWritingPath();
    ...
};
```

作用：

- 支持等待 JSON 加载完成后再绘制预览。

#### 8. `sendWriting()` 改为异步

修改前：

```js
window.sendWriting = () => {
    const { points, lift } = makeWritingPath();
    ...
};
```

```js

```

作用：

- 支持等待 JSON 加载完成后再发送 `write_path`。

### 3. 具体改动

#### 3.1 `normalizeMedians()` 返回结构调整

修改前：

```js
normalizeMedians(medians) -> strokes
```

修改后：

```js
normalizeMedians(medians) -> {
    strokes,
    bounds
}
```

其中：

- `strokes`：归一化后的笔画中心线。
- `bounds`：原始字体坐标的包围盒信息。

`bounds` 当前包含：

```js
{
    minX,
    maxX,
    minY,
    maxY,
    centerX,
    centerY,
    rangeX,
    rangeY
}
```

用途：

- 当前主要用于调试和后续扩展。
- 后续阶段可以基于它做更严格的比例控制、对齐方式和可视化标注。

#### 3.2 `getWritingStrokes()` 同步返回 `bounds`

修改后，JSON 字库路径返回：

```js
{
    strokes: normalized.strokes,
    bounds: normalized.bounds,
    source: 'hanzi-data'
}
```

回退路径仍然返回：

```js
{
    strokes: yongFallbackStrokes 或 fallbackStrokes,
    source: 'fallback'
}
```

### 4. 坐标映射逻辑调整

#### 4.1 新增 `makePose()`

新增：

```js
const makePose = (x, y, z) => ([
    Number(x.toFixed(3)),
    Number(y.toFixed(3)),
    Number(z.toFixed(3)),
    rx, ry, rz
]);
```

作用：

- 统一生成 `[X, Y, Z, RX, RY, RZ]` 格式。
- 避免每处手写重复的点格式化逻辑。

#### 4.2 新增 `mapToPaper()`

新增：

```js
const mapToPaper = (nx, ny, charCenterNormX, charScaleX) => {
    const x = originX + (charCenterNormX + nx * charScaleX) * widthMm;
    const y = originY + ny * heightMm;
    return { x, y };
};
```

作用：

- 将归一化字形坐标转换为机器人纸面坐标。
- 统一处理单字和多字横向布局的 X 方向缩放。

当前坐标约定：

- `originX`、`originY` 是书写区域中心。
- `widthMm`、`heightMm` 是整个书写区域尺寸。
- 单字情况下，归一化 `nx` 范围约为 `[-0.5, 0.5]`。
- 多字情况下，每个字在 X 方向按字符数量压缩并排列。

#### 4.3 新增 `appendStroke()`

新增：

```js
const appendStroke = (stroke, charCenterNormX, charScaleX) => {
    ...
};
```

作用：

- 将单笔归一化 stroke 转成实际机器人航点。
- 每一笔自动插入：

```text
起点抬笔
起点落笔
笔画书写点
终点抬笔
```

这样能保证不同笔画之间不会被当成连续线段。

### 5. 多字基础布局调整

原逻辑中使用：

```js
charAdvance = widthMm / chars.length
xOffset = ...
xScale = ...
```

现改为：

```js
const charCount = Math.max(1, chars.length);
const charCenterNormX = (ci - (charCount - 1) / 2) / charCount;
const charScaleX = 1 / charCount;
```

作用：

- 让每个字分配到整体书写宽度中的一个横向槽位。
- 仍然保持最多 3 个字的现有限制。
- 为后续阶段的完整多字排版打基础。

注意：

- 当前仍只是基础横排。
- 字距、换行、整体缩放策略尚未完善，属于后续任务。

### 6. 预览逻辑调整

`makeWritingPath()` 现在返回：

```js
{
    points,
    lift,
    raster,
    originX,
    originY,
    widthMm,
    heightMm,
    paperZ
}
```

`previewWriting()` 改为使用 `makeWritingPath()` 返回的同一组参数绘图：

```js
const { points, raster, originX, originY, widthMm, heightMm, paperZ } = await makeWritingPath();
```

预览坐标转换改为：

```js
const x = ((p[0] - originX) / widthMm + 0.5) * preview.width;
const y = (0.5 - (p[1] - originY) / heightMm) * preview.height;
```

作用：

- 避免预览使用输入框重新读取参数导致和实际发送点不一致。
- 预览和发送使用同一套 `points`。

### 9. 风险和后续处理

#### 9.1 Y 轴方向可能需要现场校准

当前映射使用：

```js
const y = originY + ny * heightMm;
```

预览使用：

```js
0.5 - (p[1] - originY) / heightMm
```

这能保证网页预览按常见屏幕坐标显示，但机器人实际纸面方向仍需结合现场坐标系确认。

如果实机写出来上下颠倒，应优先改 `mapToPaper()` 中的 Y 映射：

```js
const y = originY - ny * heightMm;
```

#### 9.2 当前仍未处理点数上限

C++ 仍存在 80 点限制。复杂字可能仍被截断。

下一步应进入点数控制与重采样：

```text
去重复点
按距离重采样
限制总点数
预览显示实际发送点数
```

#### 9.3 当前多字排版仍是基础版本

虽然代码已支持最多 3 个字横排，但还没有完整处理：

- 字距。
- 单字宽高比例。
- 多字整体居中细节。
- 超出区域拒绝。

这些不影响单字 MVP，但会影响短文本书写效果。

解析规则：

- `count > 上限` 时拒绝，且不再继续解析，避免超大 count 导致内存分配风险。
- 声明点数和实际点数不一致时拒绝。

### 4. 路径执行增强

- 任一点超出安全范围时拒绝。
- 任一点逆解或关节跳变检查失败时拒绝。
- 插补队列超过上限时拒绝。
- 成功后一次性写入底层轨迹队列。

效果：

- 从 `9002` 打开主页面时，优先走同源接口。
